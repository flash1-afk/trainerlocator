## рҹҺҜ Key Points

1. **Frontend (Dashboard.jsx)** makes requests
2. **Middleware (auth.js)** checks security FIRST
3. **Routes (users.js)** handle the request
4. **Model (User.js)** defines data structure
5. **Database (MongoDB)** stores the data

**Every request follows this path:**
```
Browser вҶ’ Server вҶ’ Middleware вҶ’ Route вҶ’ Model вҶ’ Database
                вҶҗ Response вҶҗв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҖвҶҗв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
```

## рҹӣЎпёҸ Security Layers

```
Layer 1: Browser
  в””в”Җв”Җв”Җ Token stored in localStorage
       (Only accessible by same origin)

Layer 2: HTTPS
  в””в”Җв”Җв”Җ Encrypted connection
       (Data can't be intercepted)

Layer 3: Auth Middleware
  в””в”Җв”Җв”Җ Checks token on every request
       (Blocks unauthorized access)

Layer 4: Route Permissions
  в””в”Җв”Җв”Җ Checks if user can do action
       (Owner or admin only)

Layer 5: Database
  в””в”Җв”Җв”Җ Password never sent back
       (Hashed and secure)
```

## рҹ“Ҡ Data Flow Summary

```
CREATE USER:
Frontend вҶ’ POST /api/auth/register
         вҶ’ Server creates user
         вҶ’ Database saves user
         вҶ’ Server sends token
         вҶ’ Frontend stores token

VIEW PROFILE:
Frontend вҶ’ GET /api/users/:id (with token)
         вҶ’ Auth checks token
         вҶ’ Route gets user from database
         вҶ’ Server sends user (no password)
         вҶ’ Frontend displays user

UPDATE PROFILE:
Frontend вҶ’ PUT /api/users/:id (with token + data)
         вҶ’ Auth checks token
         вҶ’ Route validates data
         вҶ’ Route updates user in database
         вҶ’ Server sends updated user
         вҶ’ Frontend updates display

BOOK SESSION:
Frontend вҶ’ POST /api/bookings (with token + booking data)
         вҶ’ Auth checks token
         вҶ’ Route validates booking
         вҶ’ Route creates booking in database
         вҶ’ Server sends confirmation
         вҶ’ Frontend shows success message
```
